# typeof & instanceof 区别

# typeof

typeof 能校验六中原始类型 undefined null Number String Symbol Boolean; 但是 typeof null 输出的是 object

console.log(typeof undefined);
console.log(typeof null)

typeof 一个引用类型 输出 object typeof function 输出 function

console.log(typeof []);
console.log(typeof new RegExp('ddf'));
console.log(typeof function name(params) {});
// 输出
// object
// object
// function

# Object.prototype.toString

Object.prototype.toString 是另外一种数据类型校验的方法。可以完整校验出数据类型

console.log(Object.prototype.toString.call([]));
console.log(Object.prototype.toString.call(new RegExp('ddf')));
console.log(Object.prototype.toString.call(function name(params) {}));
console.log(Object.prototype.toString.call(1));
// 输出
// [object Array]
// [object RegExp]
// [object Function]
// [object Number]

Object.prototype.toString 的缺点是：只能校验已有的数据类型，对于自定义的数据类型无法校验。例如下面的例子

class A {}
let a = new A();
console.log(Object.prototype.toString.call(a));
// 输出
// [object Object]

无法校验 a 是 A 类型。只能使用 instanceof 校验

# instanceof

校验类型 instancof 的使用:

console.log([] instanceof Array);
console.log([] instanceof Object);
// 输出
// true
// true

instanceof 的原理, __proto__ 等于原型 prototype

console.log([].__proto__ === Array.prototype);
console.log([].__proto__.__proto__ === Object.prototype);
