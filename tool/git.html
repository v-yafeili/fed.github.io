# git 提交规范

  • feat: 新增功能。

  • fix: 修复 bug。

  • docs: 文档相关的改动。

  • style: 对代码的格式化改动，代码逻辑并未产生任何变化。

  • test: 新增或修改测试用例。

  • refactor: 重构代码或其他优化举措。

  • chore: 项目工程方面的改动，代码逻辑并未产生任何变化。
